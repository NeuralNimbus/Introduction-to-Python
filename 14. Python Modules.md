# Python Modules

## What is a module?
A module is a Python file (`.py`) that contains reusable code such as functions, classes, and constants.

Using modules helps keep code organized and avoids repetition.

## Why modular code is important

- Better readability
- Easy testing and reuse
- Cleaner interview code when problem grows

## Import styles

```python
import math
print(math.sqrt(16))

from math import pi
print(pi)

import math as m
print(m.factorial(5))
```

Best practice: prefer explicit imports and avoid `from module import *`.

## Creating your own module

`utils.py`

```python
def is_even(n):
    return n % 2 == 0
```

`main.py`

```python
import utils
print(utils.is_even(8))
```

## `__name__ == "__main__"`
This checks whether a file is being run directly or imported.

```python
def run():
    print("running")

if __name__ == "__main__":
    run()
```

## Useful standard modules for interviews

- `math`: math functions
- `collections`: `Counter`, `defaultdict`, `deque`
- `heapq`: priority queue/min-heap
- `itertools`: combinations, permutations, iterators

```python
from collections import Counter
print(Counter("banana"))
```

## Interview pitfalls

- Naming your file `math.py`, `random.py`, etc. (module shadowing)
- Wildcard imports
- Circular imports in larger code
