# Python Execution and Fundamentals

## What this chapter teaches
Before writing algorithms, you should understand how Python runs code and stores data. This helps you avoid many beginner mistakes.

## How Python executes code
When you run a Python file:

1. Python first checks syntax.
2. It compiles code into bytecode.
3. The Python Virtual Machine executes that bytecode.

So Python is interpreted at runtime, but compilation to bytecode still happens internally.

## Variables and object references
In Python, a variable stores a reference to an object, not the raw value itself.

```python
x = 10
x = "ten"  # valid because x can point to a different object
```

This is called dynamic typing.

## Mutable and immutable objects
This is one of the most important concepts for interviews.

- Immutable objects cannot be changed after creation: `int`, `float`, `bool`, `str`, `tuple`
- Mutable objects can be changed in place: `list`, `dict`, `set`

```python
a = [1, 2]
b = a
b.append(3)
print(a)  # [1, 2, 3]
```

Because `a` and `b` point to the same list, both reflect the change.

## Equality vs identity

- `==` compares values
- `is` compares memory identity (same object or not)

```python
a = [1, 2]
b = [1, 2]
print(a == b)  # True (same value)
print(a is b)  # False (different objects)
```

Use `is` mainly for `None` checks.

## Complexity mindset (interview habit)
You should always think: "How fast is this operation?"

- List append: amortized O(1)
- Dict lookup: average O(1)
- Set membership: average O(1)
- Sorting: O(n log n)

## Interview pitfalls

- Using `is` instead of `==` for normal value comparison
- Forgetting mutable side effects
- Ignoring time complexity while choosing data structures
