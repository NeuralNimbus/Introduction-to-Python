# Getting Started with Python

## What is a variable?
A variable is a name that refers to a value stored in memory.  
It lets us reuse data instead of writing the same value again and again.

```python
name = "Riya"
age = 22
```

## Identifiers (naming rules)
Identifiers are names for variables, functions, classes, etc.

Rules:

- Use letters, digits, and `_`
- Cannot start with a digit
- Cannot be a Python keyword like `if`, `for`, `class`

Best practice: use `snake_case` names (`total_marks`, `is_valid`).

## Python keywords
Keywords have special meaning in Python grammar and cannot be used as custom names.

Example: `if`, `else`, `for`, `while`, `def`, `class`, `return`, `True`, `False`, `None`.

## Literals and data types
Literals are fixed values written directly in code.

- Numeric: `10`, `3.14`
- String: `"hello"`
- Boolean: `True`, `False`
- Special: `None`

```python
a = 10
b = 3.14
message = "hello"
ready = True
value = None
```

## Type conversion
`input()` gives strings, so conversion is common.

```python
s = "123"
n = int(s)
print(n + 1)  # 124
```

## Statements and indentation
Python uses indentation to define code blocks.

```python
score = 80
if score >= 40:
    print("Pass")
else:
    print("Fail")
```

Use 4 spaces consistently.

## Comments and docstrings
Comments explain logic for humans.

- `#` for single-line comments
- Triple-quoted docstrings for functions/classes/modules

```python
def add(a, b):
    """Return sum of two numbers."""
    return a + b
```

## Interview pitfalls

- Mixing tabs and spaces in indentation
- Confusing `None` with `0` or empty string
- Naming variables with keywords
