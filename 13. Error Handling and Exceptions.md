# Error Handling and Exceptions

## Why exceptions matter
Real-world and interview code should handle invalid cases gracefully, not crash unexpectedly.

## What is an exception?
An exception is a runtime error that interrupts normal program flow.

Examples: invalid integer conversion, missing key, division by zero.

## Basic `try-except`

```python
try:
    x = int(input("Enter int: "))
except ValueError:
    print("Invalid integer")
```

Put risky code in `try`, and recovery logic in `except`.

## `else` and `finally`

```python
try:
    x = 10 / 2
except ZeroDivisionError:
    print("division by zero")
else:
    print("success", x)
finally:
    print("always executes")
```

- `else`: runs if no exception occurs
- `finally`: always runs (cleanup code)

## Raising exceptions manually

```python
def set_age(age):
    if age < 0:
        raise ValueError("age cannot be negative")
```

Use `raise` when input violates function rules.

## Common built-in exceptions

- `ValueError`
- `TypeError`
- `KeyError`
- `IndexError`
- `ZeroDivisionError`

## Interview best practices

- Catch specific exceptions, not broad `Exception`
- Keep error messages clear
- Validate input early

## Interview pitfalls

- `except: pass` hiding real bugs
- Using exceptions for normal control flow
- Ignoring edge cases in parsing
