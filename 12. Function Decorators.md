# Function Decorators

## Why decorators are important
A decorator lets you add extra behavior to a function without changing the original function body.  
This is useful for logging, timing, access checks, retries, and caching.

In interviews, decorators are used to test your understanding of higher-order functions and closures.

## Core idea
A decorator is a function that takes another function and returns a new function.

```python
def decorator(func):
    def wrapper(*args, **kwargs):
        # extra behavior before
        result = func(*args, **kwargs)
        # extra behavior after
        return result
    return wrapper
```

`@decorator` is syntactic sugar for:

```python
my_func = decorator(my_func)
```

## Basic example

```python
def announce(func):
    def wrapper(*args, **kwargs):
        print("Function started")
        result = func(*args, **kwargs)
        print("Function ended")
        return result
    return wrapper

@announce
def greet(name):
    print(f"Hello, {name}")

greet("Asha")
```

## Important best practice: `functools.wraps`
Without `@wraps`, function metadata like name/docstring is lost.

```python
from functools import wraps

def announce(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        print("start")
        return func(*args, **kwargs)
    return wrapper
```

## Decorator with arguments
Sometimes the decorator itself needs input.

```python
from functools import wraps

def repeat(times):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            for _ in range(times):
                func(*args, **kwargs)
        return wrapper
    return decorator

@repeat(3)
def say_hi():
    print("Hi")
```

## Common built-in decorators

- `@staticmethod`: method that does not use instance (`self`) or class (`cls`)
- `@classmethod`: method that receives class as first argument (`cls`)
- `@property`: access method like an attribute

## Interview pitfalls

- Forgetting to return `wrapper` from decorator
- Not forwarding `*args, **kwargs`
- Missing `functools.wraps`
- Confusing decorator syntax with function call order

## Quick practice

1. Write a decorator that measures execution time.
2. Write a decorator that prints function arguments.
3. Write a decorator that blocks negative inputs.
