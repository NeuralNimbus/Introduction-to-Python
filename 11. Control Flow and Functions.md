# Control Flow and Functions

## Why this chapter matters
Control flow and functions are the foundation of every coding interview solution.

## Conditional statements
Use `if/elif/else` to execute code based on conditions.

```python
x = 7
if x % 2 == 0:
    print("even")
elif x % 3 == 0:
    print("divisible by 3")
else:
    print("other")
```

## Loops

- `for` is used when iterating over a sequence/range
- `while` is used when repeating until a condition changes

```python
for i in range(3):
    print(i)

n = 3
while n > 0:
    n -= 1
```

Use `break` to stop early and `continue` to skip current iteration.

## Functions
Functions make code reusable and readable.

```python
def add(a, b=0):
    return a + b
```

- Parameters receive input
- `return` sends output back
- Default arguments are used when caller skips a value

## Variable-length arguments

```python
def total(*nums):
    return sum(nums)

def show(**info):
    return info
```

- `*args` collects positional arguments
- `**kwargs` collects named arguments

## Scope

- Local variables exist inside functions
- Global variables exist at module level

## Common bug: mutable default argument
Default values are evaluated once, at function definition time.

Bad:

```python
def append_item(x, bucket=[]):
    bucket.append(x)
    return bucket
```

Good:

```python
def append_item(x, bucket=None):
    if bucket is None:
        bucket = []
    bucket.append(x)
    return bucket
```

## Interview pitfalls

- Forgetting to return a value
- Writing very long functions instead of helper functions
- Not discussing time complexity of loops/functions
